（二） 系统结构设计
这个博客系统采用 B/S（浏览器/服务器）架构，把主要业务逻辑放在服务端处理。这样做的好处是不用在不同设备上做太多适配，服务端更新时也更方便。用户不用安装专门的软件，打开浏览器就能访问。整个系统从逻辑上分成三层：前端展示、后端处理和数据存储。

前端部分是基于 Vue 3 开发的单页面应用，在浏览器里运行。它负责显示界面和处理用户操作，通过 HTTP 请求和后端交换数据。技术上用了 Vue 3 的 Composition API，配合 Pinia 管理状态，Vue Router 处理页面跳转，还有 Element Plus 组件库来构建界面，这样用户体验更流畅，界面也能适配不同屏幕尺寸。

后端是用 Spring Boot 框架搭建的，内置了 Web 服务器来接收请求。请求先到 Controller 层，再转发到 Service 层处理具体业务，比如文章管理、评论处理和用户登录这些功能。为了提高性能，系统加了 Redis 缓存；异步任务用 RabbitMQ 消息队列来处理；用户登录认证则用 JWT 令牌，这样服务端不用保存会话状态，更灵活。

数据存储用的是 MySQL 数据库，通过 MyBatis-Plus 把代码里的对象和数据库表关联起来。数据库设计包含文章、标签、分类、评论、用户等核心表，支持复杂的关联查询和事务处理。

系统采用前后端分离的开发模式，后端不再直接返回整页 HTML，而是通过 RESTful API 发回 JSON 数据，由前端框架动态渲染页面。这种分层设计让前后端职责更清晰，合作更顺畅，也让系统更稳定，以后扩展功能时也更方便。

（三） 系统流程设计
下面重点说几个关键流程。

1. 用户注册与登录
新用户点注册按钮，填好用户名、密码、邮箱这些信息提交。系统在后台检查信息是否合法，没问题就创建账号。用户用账号密码登录时，系统验证通过后会生成 JWT 令牌，前端把令牌存起来，以后发请求时带着它证明身份。如果密码输错了，系统会提示重新输入，还提供找回密码的入口，确保用户账号安全。

2. 文章管理
用户进入发布文章页面，填标题、写内容、选分类和标签，上传封面图后提交。系统在后台检查内容，处理格式（比如把 Markdown 转换成 HTML），然后存到数据库。发布后的文章会在前台显示，用户可以按分类、标签或关键词搜索。文章详情页支持评论、点赞和收藏，这些操作的数据会实时更新。

3. 评论与互动
用户在文章详情页可以写评论提交，系统验证用户身份后保存评论并更新评论列表。评论支持回复，形成嵌套结构。用户还能给评论点赞，系统会记录点赞状态并更新数量。管理员在后台可以管理评论，删除或屏蔽违规内容。

4. 后台管理
管理员登录后台后，进入内容管理中心。在文章管理模块，可以发布、编辑、删除文章，设置文章状态（草稿、已发布、下架）。用户管理模块里，可以查看用户列表、编辑用户信息、分配角色权限。系统设置模块可以配置网站基本信息、管理分类和标签、查看系统操作日志。所有操作都会记录日志，方便追溯和保障安全。

5. 标签与分类管理
管理员在后台可以创建、编辑、删除文章分类和标签。分类是层级结构，支持多级管理；标签是扁平的，创建后可以自由关联到文章。前端根据分类和标签展示文章列表，用户点分类或标签就能快速找到相关内容，提高浏览效率。
